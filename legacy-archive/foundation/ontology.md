# Liahona Ontology

## Core Distinction

Liahona.AI is not merely a chatbot or application.

It is an orienting interpretive system grounded in canonical truth and sustained through meaningful continuity across time.

The system is organized into five primary layers:

Canonical
Interpretive
Runtime
Projection
Continuity

These layers should remain distinct.

The purpose of this distinction is:
- epistemic clarity
- humility
- truthful grounding
- continuity coherence
- architectural restraint

---

# Canonical Layer

The canonical layer contains authoritative grounding sources external to the system itself.

Examples:
- Bible
- Book of Mormon
- Doctrine and Covenants
- Pearl of Great Price
- Gospel Library
- Guide to the Scriptures
- sourced doctrinal references

Canonical sources remain:
- outward-facing
- authoritative outside the system
- non-modified
- clearly sourced

Liahona is not the authority.
It points toward authoritative grounding.

---

# Interpretive Layer

The interpretive layer contains:
- symbolic meaning
- experiential philosophy
- artifact metaphors
- cosmology
- orientation systems
- continuity interpretation
- architectural symbolism

Interpretive systems are:
- human-built
- symbolic
- experiential
- reflective

Interpretation must never pretend to be canon.

---

# Runtime Layer

The runtime layer contains:
- code
- rendering systems
- memory systems
- interaction systems
- databases
- deployment infrastructure
- APIs
- operational tooling

Examples:
- Node.js
- PM2
- embeddings
- semantic retrieval
- Discord integration
- localStorage
- hosting infrastructure

Runtime systems are delivery mechanisms, not foundational truth.

---

# Projection Layer

The projection layer surfaces meaning into experience.

Examples:
- the artifact interface
- scripture grounding
- continuity logs
- fragments
- source cards
- reflective dashboards
- symbolic interaction

Projection translates deeper architecture into human-facing orientation.

---

# Continuity Layer

The continuity layer preserves meaningful evolution across time.

It preserves:
- foundational shifts
- architectural decisions
- governance refinements
- major experiential transitions
- meaningful checkpoints

It should avoid:
- noise
- excessive logs
- meaningless operational clutter
- preservation without meaning

Continuity should remain:
- semantic
- restrained
- readable
- meaningful

# Governing Principle

Orientation over engagement.

The system should prioritize:
- truth
- humility
- clarity
- restraint
- continuity
- reflection
- meaningfulness

Over:
- stimulation
- addiction
- retention optimization
