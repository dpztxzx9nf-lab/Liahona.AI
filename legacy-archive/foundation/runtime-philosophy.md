# Runtime Philosophy

## Runtime Exists To Serve Orientation

Technology is infrastructure.

Infrastructure supports:
- continuity
- orientation
- retrieval
- interaction
- projection

Infrastructure is not the meaning itself.

---

# Runtime Principles

Prefer:
- modularity
- clarity
- maintainability
- semantic organization
- restrained complexity
- rebuildability

Avoid:
- unnecessary abstraction
- architectural bloat
- feature accumulation
- operational chaos

---

# Development Order

1. Establish modular structure
2. Establish continuity foundation
3. Keep repositories clean
4. Keep Git disciplined
5. Then iterate aggressively

---

# Core Insight

The project should remain rebuildable.

KINDEX is durable human continuity.

Liahona is an evolving interpretive layer around it.

The runtime may evolve repeatedly without losing the deeper continuity structure.
