# Dev Portal

This folder is intended for operational continuity memory: deployment notes, testing observations, failures, logs, architecture notes, runtime observations, and development journal entries.

At the moment, most files in this folder are placeholders. Treat them as category markers unless they are populated with dated, specific operational content.

## Intended Use

Use `dev-portal/` for practical development memory that helps future work:
- what was deployed
- what was tested
- what failed
- what changed operationally
- what runtime behavior was observed
- what should be checked next

## Boundary

This folder is not active governance.

Use `core/` for governance and ontology.

Use `continuity/` for curated semantic checkpoints.

Use `dev-portal/` for early operational notes only when they are concrete enough to be useful.
