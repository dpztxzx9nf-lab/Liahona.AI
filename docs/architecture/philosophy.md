# Architecture Philosophy

Liahona.AI is an artifact-first interface for Liahona and KINDEX continuity.

The front end should prioritize:

- The Liahona object as the primary orienting anchor.
- Atmosphere, depth, restraint, and symbolic discovery.
- Lightweight browser-native interaction.
- A continuity device as the secondary operational layer.

The operational layer should hold project memory, logs, return points, source references, and later backend-connected continuity.

Current boundaries:

- Static GitHub Pages site.
- No backend.
- No authentication.
- No heavy 3D engine.
- No multiplayer.
- Local browser state is stored with `localStorage`.

Future backend, login, retrieval, and multiplayer systems should attach behind the continuity layer without making the first-load experience feel like a dashboard.
