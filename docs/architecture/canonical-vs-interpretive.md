# Canonical vs Interpretive Architecture

Liahona.AI should keep clear boundaries between source material, interpretation, runtime behavior, projection, and continuity.

## Canonical Layer

The canonical layer is the source layer.

It includes scripture, official source references, and other grounding material that should be pointed to rather than impersonated or absorbed into authority claims.

Canonical sources are not rewritten by Liahona. They remain outside the system as references.

## Interpretive Layer

The interpretive layer is where Liahona makes restrained judgments about context, direction, timing, and relevance.

This layer can summarize, connect, and orient, but it should remain humble and clearly human-built.

It should not pretend to be a canonical source or speak with divine authority.

## Runtime Layer

The runtime layer is the actual behavior of the site or bot.

It includes UI state, Discord behavior, delivery decisions, logging, localStorage, and any future retrieval or generation flow.

Runtime behavior should be observable, reversible where practical, and honest about what is implemented.

## Projection Layer

The projection layer is how meaning appears to the user.

It includes the Liahona artifact, motion, light, visual atmosphere, the continuity device, and other symbolic interface surfaces.

Projection should support orientation without becoming theatrical roleplay or visual clutter.

## Continuity Layer

The continuity layer preserves meaningful project memory over time.

It includes logs, versions, decisions, return points, source plans, and durable architectural notes.

Continuity should record meaningful changes and directional transitions, not every minor fluctuation.

## Why This Matters

This distinction protects humility by keeping Liahona from presenting itself as the source of authority.

It protects clarity by making each layer's responsibility easier to understand.

It protects sourcefulness by pointing outward to references instead of replacing them.

It protects architectural honesty by separating what the system means, what it does, what it displays, and what it remembers.
